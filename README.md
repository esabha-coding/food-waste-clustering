# Food Waste Clustering

## Project Overview

This project analyzes global food-wastage data, applies preprocessing and unsupervised clustering (K-Means), and provides a minimal FastAPI backend plus a Next.js frontend to explore cluster summaries and visualizations.

## Key Features

- Data cleaning and numeric conversion for raw CSV values
- Feature selection and standard scaling for clustering
- K-Means clustering with silhouette-based guidance
- Cluster summary statistics, top countries/food-types per cluster, and visualizations (boxplots, timelines)
- FastAPI backend serving cluster summaries and endpoints
- Next.js frontend that fetches and displays analysis results

## Technology Stack

- Python 3.14 — data processing and backend
- pandas, numpy, scikit-learn, matplotlib — data science stack
- FastAPI + Uvicorn — backend API
- Node.js, Next.js (React) — frontend
- Git + GitHub — version control and hosting

## Project Architecture

The repository follows a simple, single-repo layout where the backend and frontend are colocated. Data processing scripts produce CSV artifacts consumed by the backend which exposes HTTP endpoints. The frontend calls these endpoints to render summaries and charts.

Flow:

1. `food_waste_analysis.py` ingests and preprocesses `WorldWide_foodwastage_dataset.csv`, runs K-Means and writes `food_waste_clustered.csv`.
2. `cluster_analysis.py` computes descriptive statistics and writes visualization PNGs.
3. `backend/app.py` reads `food_waste_clustered.csv` and serves summary endpoints.
4. `frontend/` fetches `/api/summary` (proxied) and presents UI.

## Installation Guide

Prerequisites:

- Python 3.10+ (3.14 used in development)
- Node.js 18+ and npm

Steps (local dev):

1. Create and activate a Python virtual environment (recommended):

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
```

2. Install backend requirements:

```powershell
python -m pip install -r backend/requirements.txt
```

3. Install frontend dependencies (from `frontend/`):

```bash
cd frontend
npm install
```

## Running the Project

1. Start the backend (from repo root):

```bash
python -m uvicorn backend.app:app --reload --host 127.0.0.1 --port 8000
```

2. Start the frontend (in a separate terminal, from `frontend/`):

```bash
npm run dev
# open http://localhost:3000
```

The frontend proxies API calls to the backend; ensure the backend is running first.

## API Endpoints

- `GET /health` — health check (if implemented)

## Customer Segments Classification

This project clusters records in the dataset (e.g., by country, commodity, or waste cause) and produces a simple segmentation useful for:

- Policymakers looking to prioritize interventions by cluster characteristics
- NGOs and researchers comparing waste patterns across regions
- Data analysts using clusters as labels for downstream supervised models

The clustering produced here is unsupervised — interpret clusters with domain knowledge before using them in production.
